Kroki które claudei init  ulepszony polecił


1. Fix CI branch mismatch — .github/workflows/ci.yml triggers on master but your default branch is main. Update the trigger to main so CI actually fires on pushes.
2. Install GitHub CLI — sudo dnf install gh && gh auth login (see above). Enables Claude to open PRs, check CI status, and manage issues directly.
3. Install the Playwright plugin — this project has a React frontend. /plugin install playwright@claude-plugins-official lets Claude launch a real browser, screenshot pages, and fix visual bugs itself.
4. Install the frontend design plugin — /plugin install frontend-design@claude-plugins-official gives Claude design principles and component patterns for polished Astro + shadcn/ui work.
5. Add a test framework — no test runner is configured. Consider Vitest (fits well with the Vite/Astro stack) so Claude can verify its own changes.
6. Create and optimize skills — /plugin install skill-creator@claude-plugins-official then /skill-creator <name> to build project-specific skills (e.g. a /deploy-preview or /verify-auth workflow).
7. Browse official plugins — run /plugin to see the full catalog of bundled skills, agents, hooks, and MCP servers.
